# Project-Labeled Category-King Documentation

This section centralizes **project-labeled** documentation in one place, grouped by discipline and named per project (for example: `CRM_Figma_*`, `CRM_Architecture_*`).

## Folder Structure

- `strategy/`: Product, workflow, and business execution narratives by project.
- `architecture/`: System/Service architecture blueprints by project.
- `figma/`: Extensive Figma Make prompt packs designed for category-king UX outcomes.
- Shared playbook: [Portfolio Figma Category-King Playbook](figma/Portfolio_Figma_Category_King_Playbook.md)

## Naming Convention

- Strategy: `<PROJECT>_Strategy_Category_King_Blueprint.md`
- Architecture: `<PROJECT>_Architecture_Category_King_Blueprint.md`
- Figma: `<PROJECT>_Figma_Category_King_Prompts.md`
